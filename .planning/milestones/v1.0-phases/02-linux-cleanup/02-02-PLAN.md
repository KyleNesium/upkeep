<objective>
Add Linux user-cache and systemd journal cleanup to cleandeep. Covers LNX-03 (user cache) and LNX-04 (systemd journal).

Purpose: On Linux, the two largest reclaimable storage pools outside the package manager are `~/.cache/` (per-user app caches) and the systemd journal (`/var/log/journal`). Phase 3 "Dev Tool Caches" already scans `~/.cache/*/` on all platforms but needs a Linux-specific approval flow for removal. Phase 9 "Stale Logs" is macOS-centric (`~/Library/Logs`) and needs a Linux branch for journalctl. Both use approval gates and never sudo — the user journal vacuums without sudo; the system journal is surfaced as a manual step.

Output: Phase 3 shows ~/.cache/*/ top consumers and lets the user remove specific dirs on Linux; Phase 9 reports journal disk usage and vacuums the user journal on approval; system journal cleanup is surfaced as a sudo command the user must run manually.
</objective>

```bash
if [ "$OS_TYPE" = "linux" ] || [ "$OS_TYPE" = "wsl2" ]; then
  echo "=== Total ~/.cache size ==="
  du -sh ~/.cache/ 2>/dev/null || echo "~/.cache/ not present"
  echo "=== Top 15 subdirectories (sorted largest first) ==="
  du -sh ~/.cache/*/ 2>/dev/null | sort -rh | head -15

```

> **Linux cache removal — approval gate.** If the output above shows any subdirectory over 100MB, present it as a numbered table (index, path, size) and prompt: "Remove which? NEVER clear `~/.cache/` as a whole directory — some tools (e.g. `~/.cache/mesa_shader_cache`, `~/.cache/fontconfig`) rebuild slowly and impact desktop startup. Only remove named subdirectories the user approves. Known-safe-to-remove subdirs: `pip`, `yarn`, `pnpm`, `go-build`, `thumbnails`, `mozilla`, `chromium`, `vscode-cpptools`. Known-costly-to-rebuild (warn before removing): `mesa_shader_cache`, `fontconfig`, `nvidia`.

Do NOT change Step 1 or Step 2 of Phase 3 — they remain cross-platform.
