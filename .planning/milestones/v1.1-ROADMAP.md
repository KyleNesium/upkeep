# v1.1 Roadmap — Update Skill Overhaul

| Phase | Name | Plans | Goal |
|-------|------|-------|------|
| 7 | Parallel Discovery Agents | 1 | Four specialized scouts replace sequential bash; emit JSON |
| 8 | Compatibility Synthesizer | 1 | One agent reads scout JSON, emits ordered plan with risk flags |
| 9 | Single-Gate Apply + Post-Flight | 1 | One approval gate, parallel apply where safe, brew doctor + PATH re-check |

## Phase ordering rationale

Phases must land in order because each consumes the previous phase's output:

- Phase 7 defines the JSON schema scouts emit.
- Phase 8 consumes that schema and produces the apply plan.
- Phase 9 consumes the apply plan and runs it.
