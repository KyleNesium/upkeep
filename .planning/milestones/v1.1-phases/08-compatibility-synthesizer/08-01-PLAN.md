# Phase 8 Plan — Compatibility Synthesizer

## Scope

Add a single Agent call between Phase 7 discovery and Phase 9 apply.
Author the static compatibility matrix file. Document the synthesizer's
prompt, output schema, and history-based ETA heuristic.

## Deliverables

1. New file `upkeep/skills/update/compatibility.json` (matrix).
2. New section in `SKILL.md` titled "Step 2 (macOS) — Synthesize Plan"
   gated on `$OS_TYPE = "macos"`.
3. Synthesizer agent prompt template included verbatim in SKILL.md.
4. Output schema documented inline.
5. ETA heuristic documented (history file location, defaults table).

## Tasks (atomic)

### T1 — Author `compatibility.json`

Edges listed in 08-CONTEXT.md. Each edge: `from` (semver-tagged source
formula or category), `to` (downstream category), optional `formulas`
list (specific gems/npm packages affected), human-readable `reason`.

### T2 — Author synthesizer agent prompt

Prompt structure:
- Role: "You are the upkeep compatibility synthesizer."
- Input contract: JSON discovery payload + compatibility.json contents
  pasted in full.
- Output contract: JSON plan per schema in 08-CONTEXT.md.
- Hard constraints:
  - "Only reference downstream effects that appear in compatibility.json"
  - "Classify version delta as major/minor/patch using semver — exact
    rules listed"
  - "Emit JSON only, no prose, no markdown fences"
  - "If `gems.system_ruby` is true, set
    `tool_specs.gems.command` to `gem update --user-install`"
  - "If disk free_gb < 5, set `warnings[].severity = high` and emit no
    `ordered_groups`"

### T3 — ETA heuristic

History file: `~/.claude/data/upkeep-history.json`:

```json
{
  "schema_version": "1",
  "runs": [
    {
      "ts": "2026-05-07T10:48:00Z",
      "minutes": {"brew": 11, "npm": 1, "pipx": 1, "gems": 12, "uv": 0.2, "bun": 0.2}
    }
  ]
}
```

Synthesizer reads the file (if present), takes the median of the last
five runs per category, multiplies by `(outdated_count / median_count)`
to scale.

If no history: use bake-in defaults.

| Category | Default per item | Notes |
|---|---|---|
| brew    | 25 sec/package | Heavy compiles (qt, ffmpeg) skew higher |
| npm     | 30 sec/package |  |
| pipx    | 20 sec/tool    |  |
| gems    | 15 sec/gem     | System Ruby gem dirs are slow |
| uv      | 5 sec total    | Self-update |
| bun     | 5 sec total    | Self-update |
| skills  | 3 sec/repo     | git fetch + ff-only pull |
| mas     | 30 sec/app     |  |
| macOS   | 5 min          | Excluding restart |

### T4 — History file write hook

Phase 9 (apply) writes elapsed wall time per category back to the history
file at the end. Phase 8 reads it. Self-improving over runs.

### T5 — Synthesizer fallback

If the synthesizer agent fails (timeout, schema-invalid output), fall
back to a simpler rule-based plan: skills first, brew next, then
language ecosystems serially, then mas/macOS. No compatibility flags,
no ETA. Surface "compatibility analysis unavailable — running in
fallback mode."

## Out of scope

- Self-modifying compatibility matrix (LLM proposes new edges) — too
  risky, defer.
- Network calls to release notes APIs — defer; CHANGELOG.md scan in
  Phase 7 is enough for now.
- Cross-platform compatibility edges (Linux openssl differs) — Mac-only.
