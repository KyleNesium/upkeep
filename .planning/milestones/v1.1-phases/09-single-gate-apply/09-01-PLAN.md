# Phase 9 Plan — Single-Gate Apply + Post-Flight

## Scope

Implement the apply orchestration in `SKILL.md`: single approval gate,
parallel ecosystem apply, post-flight health check, history file write,
final report.

## Deliverables

1. New section in `SKILL.md` titled "Step 3 (macOS) — Approve & Apply"
   gated on `$OS_TYPE = "macos"`.
2. New section "Step 4 (macOS) — Post-Flight" with brew doctor + PATH
   re-check + deprecation aggregation.
3. New section "Step 5 (macOS) — Report" using the existing v1.0 report
   table format with risk and downstream-effects callouts.
4. History-write logic appended to apply phase.

## Tasks (atomic)

### T1 — Approval gate

Single `AskUserQuestion` with three options (Apply all / Drop categories
/ Cancel) and a long question body containing the synthesizer's plan
summary.

If "Drop categories", emit a second multi-select question with one option
per category. Categories the user keeps move forward; the rest are
recorded as `↷ skipped (user)`.

### T2 — Apply orchestrator

Pseudocode:

```
for group in plan.ordered_groups:
  case group.parallelism:
    serial:    run each tool in order
    exclusive: run the single tool, await completion
    parallel:  fan out via run_in_background, cap 4 concurrent, await all
  for each tool result: record exit code, wall time, stdout tail
```

Implementation notes:
- Each tool command comes from `tool_specs[tool].command` in the plan.
- `tool_specs[tool].preconditions` run as a serial preface (e.g., `brew
  update` before `brew upgrade`).
- Wall time captured by wrapping the command with `time` and parsing
  stderr.

### T3 — Per-tool failure isolation

Wrap each tool command with:

```bash
if ! eval "$CMD" >>"$LOG" 2>&1; then
  echo "::result::$TOOL=fail:$?"
else
  echo "::result::$TOOL=ok"
fi
```

Aggregator parses `::result::` markers from each log to classify.

### T4 — Post-flight checks

```bash
# brew doctor noise filter
DOCTOR_OUT=$(brew doctor 2>&1)
case "$DOCTOR_OUT" in
  *"Your system is ready to brew."*) ;; # silent
  *) echo "── brew doctor ──"; echo "$DOCTOR_OUT" ;;
esac

# PATH shadow re-check, scoped to upgraded formulas
for FORMULA in $UPGRADED_FORMULAS; do
  for BIN in $(brew list "$FORMULA" 2>/dev/null | grep "/bin/" | xargs -n1 basename); do
    PATHS=$(which -a "$BIN" 2>/dev/null | sort -u)
    COUNT=$(echo "$PATHS" | wc -l | tr -d ' ')
    if [ "$COUNT" -gt 1 ]; then
      echo "shadow: $BIN — first match $(echo "$PATHS" | head -1)"
    fi
  done
done

# Resolution re-check
for TOOL in $UPGRADED_TOOLS; do
  if ! command -v "$TOOL" >/dev/null 2>&1; then
    echo "⚠ $TOOL no longer resolves on PATH"
  fi
done
```

### T5 — Deprecation aggregator

During apply, every tool's stdout gets piped through:

```bash
grep -E "(deprecated|deprecation|warning|WARN)" >> "$DEPRECATION_LOG"
```

Post-flight section reads `$DEPRECATION_LOG` and surfaces a deduplicated,
sorted list — one per line, capped at 20.

### T6 — History write

```bash
HIST_DIR="$HOME/.claude/data"
HIST_FILE="$HIST_DIR/upkeep-history.json"
mkdir -p "$HIST_DIR"

ENTRY=$(jq -n --arg ts "$(date -u +%FT%TZ)" \
  --argjson minutes "$MINUTES_JSON" \
  '{ts: $ts, minutes: $minutes}')

if [ -f "$HIST_FILE" ]; then
  jq --argjson entry "$ENTRY" '.runs += [$entry]' "$HIST_FILE" > "$HIST_FILE.tmp" \
    && mv "$HIST_FILE.tmp" "$HIST_FILE"
else
  jq -n --argjson entry "$ENTRY" '{schema_version:"1", runs:[$entry]}' > "$HIST_FILE"
fi
```

If `jq` not installed: skip history write silently and emit a one-line
note in the report ("install `jq` for ETA self-tuning").

### T7 — Final report

Then the existing per-tool ✓/↷/✗ table.
