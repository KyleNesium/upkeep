# Phase 10 Plan — Synthesizer Prompt & Context Fixes

## Scope

Three edits to close G1, G3, G5 from the v1.1 milestone audit.

## Tasks

### T1 — Wire history file read into Step 2m (G1, R10)

Before the synthesizer Agent invocation, add a bash block to load the
history file into a shell variable. Replace the prompt's third fenced
block from a literal placeholder to the variable's value (or `{}`).

```bash
HIST_FILE="$HOME/.claude/data/upkeep-history.json"
if [ -f "$HIST_FILE" ]; then
  HISTORY_JSON=$(cat "$HIST_FILE")
else
  HISTORY_JSON='{}'
fi
```

Then in the prompt, replace:
```
History (may be empty):
> ```json
> {{ paste ~/.claude/data/upkeep-history.json here, or {} if missing }}
> ```
```
with:
```
History (may be empty):
> ```json
> ${HISTORY_JSON}
> ```
```

The synthesizer's Hard Rule #6 (median-of-5) becomes load-bearing on the
second-and-later run.

### T2 — Add Hard Rule #8 for macOS restart (G3, R5)

Also update `.planning/phases/08-compatibility-synthesizer/08-CONTEXT.md`
`tool_specs` example to include a `macos` entry:

```json
"macos": {
  "kind": "store",

  "restart_required": true,
  "preconditions": []

```

### T3 — Extend Hard Rule #1 to consume severity fields (G5)

Replace existing Hard Rule #1:
> 1. Only reference downstream effects that appear in
>    `compatibility.json`. Do not invent edges.

with:
> 1. Only reference downstream effects that appear in
>    `compatibility.json` — do not invent edges. For each materialised
>    edge, read `severity_on_major` and `severity_on_minor` and copy

>    `plan.warnings[]` by severity: `high` first, then `medium`, then
>    `low`.

Update the example warning shape in `08-CONTEXT.md` to include
`severity` (already implied by the existing audit example —
make explicit).
