<objective>
Add a WSL2 environment banner to three banner-only skills: `cleanquick`, `audit`, and the top-level `upkeep` router. The banner is a one-line, informational header — "=== Running in WSL2 on Windows ===" — that fires only when `$OS_TYPE = "wsl2"` immediately after the existing Environment Detection `echo "Environment: ..."` line. Satisfies WSL-01 for three of the five skills (the remaining two, `cleandeep` and `update`, are banner-targeted in plans 03-02 and 03-03 alongside their other WSL2 work).

Purpose: Give users running inside WSL2 an unambiguous visual signal that the skill knows it is on Windows, before any phase runs. This closes the WSL-01 UX gap — Phase 1 set `$OS_TYPE="wsl2"` but nothing is visibly different from plain Linux today.

Output: Three SKILL.md files with an additional bash block in the Environment Detection section that conditionally prints the banner when WSL2 is detected.
</objective>
