# Phase 7 Plan — Parallel Discovery Agents

## Scope

Author the SKILL.md changes that orchestrate four parallel discovery agents
on macOS. Define each scout's prompt, allowed tools, and output schema.

## Deliverables

1. New section in `upkeep/skills/update/SKILL.md` titled
   "Step 1 (macOS) — Parallel Discovery" gated on `$OS_TYPE = "macos"`.
2. Documented JSON schema in the SKILL.md text (also lives in 07-CONTEXT.md).
3. Four scout prompt templates included verbatim in the SKILL.md.
4. Fallback path: if any scout returns a parsing error, skill falls back to
   the v1.0 sequential discovery for that domain only.

## Tasks (atomic)

### T1 — Add macOS-only branch in SKILL.md Step 1

After the existing OS detection block, add:

```
if $OS_TYPE = "macos":
   → run Parallel Discovery (this phase)
else:
   → run Sequential Discovery (v1.0 path, unchanged)
```

### T2 — Author skills-scout prompt

**Domain:** `~/.claude/skills/*/.git`, `~/.codex/skills/*/.git`,
`~/.claude/plugins/cache/`, the upkeep self-check.

**Allowed tools:** Bash (git, ls, cat for VERSION/CHANGELOG/plugin.json),
Read.

**Required output:** JSON `skills` block per schema. For each git skill,
include up to 5 `breaking_lines` extracted from CHANGELOG.md sections
between current and target versions (grep for `BREAKING\|Breaking`).

### T3 — Author native-scout prompt

**Domain:** brew (`brew outdated --json=v2` for structured output, fall
back to plain `brew outdated` if --json fails), `mas outdated`,
`softwareupdate -l` parsing for `[restart]`.

**Allowed tools:** Bash (brew, mas, softwareupdate).

**Required output:** JSON `native` block. For each brew formula,
classify version delta as `major` / `minor` / `patch` from the JSON
output's `current_version` and `installed_versions`.

### T4 — Author language-scout prompt

**Domain:** npm, pipx, gems, uv, bun, deno, rustup, cargo, mise.

**Allowed tools:** Bash for each tool.

**Required output:** JSON `language` block. Special-case: detect system
Ruby. Set `system_ruby: true` if `which ruby` returns `/usr/bin/ruby`
*and* `ruby --version` starts with `ruby 2.`. Set `ruby_version` from
that.

### T5 — Author shadow-scout prompt

**Domain:** PATH duplication for binaries belonging to brew formulas in
the outdated list. Broken symlinks under `/opt/homebrew/bin` and
`/usr/local/bin`.

**Allowed tools:** Bash (which -a, brew --prefix, brew list, find -L
for broken symlinks).

**Required input:** the list of outdated brew formulas (from
native-scout — so shadow-scout runs *after* native-scout completes, or
shadow-scout queries brew directly with `brew list --formula`).

**Decision:** shadow-scout runs in the same parallel batch as the others
to keep the architecture symmetric. It scopes itself by reading
`brew list --formula` then iterating binaries, accepting some redundant
work in exchange for parallelism.

### T6 — Disk-space pre-flight (inline in SKILL.md, not an agent)

```bash
FREE_GB=$(df -k / | awk 'NR==2 {print int($4/1024/1024)}')
if [ "$FREE_GB" -lt 5 ]; then
  echo "Refusing to start: < 5 GB free."; exit 1
elif [ "$FREE_GB" -lt 10 ]; then
  echo "⚠ Warning: < 10 GB free. Continuing."
fi
```

This block runs *before* the agents fan out.

### T7 — Document the JSON schema in SKILL.md

Inline the schema (or reference 07-CONTEXT.md). Schema must be visible to
the synthesizer agent in Phase 8.

## Verification

- Manual run on a populated machine. Expected: agents complete under 15s
  total wall time. Result is a single JSON payload matching schema.
- Manual run with brew not installed. Expected: native-scout returns
  `"brew": {"installed": false}` without erroring.
- Manual run with codex skills directory absent. Expected: skills-scout
  returns `"info": {"codex_skills_total": 0, "codex_skills_git": 0}`.
- Static check: SKILL.md `allowed-tools` frontmatter still permits every
  bash command used in scout prompts.

## Out of scope

- Apply phase (Phase 9).
- Synthesizer logic (Phase 8).
- Caching discovery results across runs (deferred — acceptable to re-run
  every invocation; wall time is the relevant metric, not cost).
