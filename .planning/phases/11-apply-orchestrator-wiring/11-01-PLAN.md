### T1 — Schema-version refusal gate (G4)

In Step 3m, between the synthesizer Agent invocation and the approval
gate, add a schema-version check:

```bash
if command -v jq >/dev/null 2>&1; then
  if ! jq -e '.schema_version == "1"' <<<"$PLAN_JSON" >/dev/null 2>&1; then
    echo "compatibility analysis unavailable — running in fallback mode"
    # Use synthesizer fallback: skills → brew → language serial → mas → macOS
    PLAN_JSON="$FALLBACK_PLAN_JSON"
  fi
fi
# If jq is missing, skip the check (graceful degradation, matches Step 5m)
```

Where `$FALLBACK_PLAN_JSON` is the rule-based plan documented at
SKILL.md line ~411 (`### Synthesizer fallback`). The fallback string
("compatibility analysis unavailable — running in fallback mode") is
the existing one — do not introduce a new variant.

### T2 — Populate `$UPGRADED_FORMULAS` / `$UPGRADED_TOOLS` (G2, R7)

Use temp files to safely accumulate from parallel apply jobs. In Step
3m's apply orchestrator preamble:

```bash
UPGRADED_FORMULAS_FILE=$(mktemp)
UPGRADED_TOOLS_FILE=$(mktemp)
trap "rm -f $UPGRADED_FORMULAS_FILE $UPGRADED_TOOLS_FILE" EXIT
```

In the per-tool isolation wrapper (currently line ~470), on success,
append the tool/formula name. Brew is a special case — on success, list
all formulas it touched:

```bash
if eval "$CMD" >>"$LOG" 2>&1; then
  RESULT="ok"
  case "$TOOL" in
    brew)
      # Append every formula that brew upgraded this run
      grep -E "^==> Upgrading" "$LOG" | awk '{print $3}' | cut -d/ -f1 \
        >>"$UPGRADED_FORMULAS_FILE"
      ;;
    *)
      echo "$TOOL" >>"$UPGRADED_TOOLS_FILE"
      ;;
  esac
else
  RESULT="fail:$?"
fi
```

Before Step 4m runs, read the temp files into the env vars Step 4m
expects:

```bash
UPGRADED_FORMULAS=$(cat "$UPGRADED_FORMULAS_FILE" | sort -u | tr '\n' ' ')
UPGRADED_TOOLS=$(cat "$UPGRADED_TOOLS_FILE" | sort -u | tr '\n' ' ')
export UPGRADED_FORMULAS UPGRADED_TOOLS
```

Step 4m's existing loops (lines 499–516) consume these correctly once
populated.
