## Gaps detail

### G1 — R10 history reader unwired (medium)

Step 2m's third input block:
```
History (may be empty):
{{ paste ~/.claude/data/upkeep-history.json here, or {} if missing }}
```

is a literal placeholder. No bash precedes Step 2m to read the file into
a variable, so the synthesizer always falls back to bake-in defaults.
R10 functional but not self-tuning.

**Fix:** add a pre-Step-2m bash snippet reading the history file into a
shell var, and change the prompt fenced block to use that var instead of
the placeholder.

### G2 — R7 `$UPGRADED_FORMULAS` / `$UPGRADED_TOOLS` undefined (medium)

Step 4m loops iterate these env vars but Step 3m's apply orchestrator
never populates them. Runtime: silent no-op.

**Fix:** in the Step 3m isolation wrapper (line 470), append the tool's
formula/name to `$UPGRADED_FORMULAS` (brew) or `$UPGRADED_TOOLS`
(everything else) on success.

### G3 — synthesizer field for macOS restart unspecified (low)

Step 3m line 477 reads `plan.tool_specs.macos.restart_required` but the
synthesizer Hard Rules don't tell it to set that field from
`native.softwareupdate.restart_required`. `08-CONTEXT.md` `tool_specs`
example doesn't include a `macos` entry.

**Fix:** add a Hard Rule #8 to the synthesizer prompt and update
08-CONTEXT.md schema example.

### G4 — schema_version refusal not implemented (low)

`08-CONTEXT.md` says "Phase 9 refuses to apply" on schema mismatch but
Step 3m has no `jq -e '.schema_version == "1"'` check.

**Fix:** add a one-line check before the approval gate; reuse the
existing fallback string ("compatibility analysis unavailable — running
in fallback mode").

### G5 — severity fields in compatibility.json unused (low)

Edges carry `severity_on_major` and `severity_on_minor`. Synthesizer
prompt never references them.

**Fix (optional):** extend Hard Rule #1 to "use severity to order/group
risk flags."
